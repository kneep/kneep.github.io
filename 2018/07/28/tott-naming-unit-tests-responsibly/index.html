好处

这些测试方法描述了对象的功能，而不是公开的接口和输入输出。将来的工程师能更容易地知道这些方法是干什么用的，不需要去研究代码。

这样的命名习惯能给你预警。例如，测试方法名前面加上类名无法构成一句通顺的句子，则说明这个测试方法放错位置了。说不清楚功能的类，一般要分解成更小更清晰的类。
